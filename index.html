解释一下Hexo常用的几个命令：

  1. $ hexo g(hexo generate) #生成静态文件，会在当前目录下生成一个新的叫做public的文件夹;
  2. $ hexo s(hexo server) #启动本地web服务，用于博客的预览,默认地址为http://localhost:4000/;
  3. $ hexo d(hexo deploy) #部署博客到远端（比如github等平台）
  4. $ hexo n "postName" #新建文章
  5. $ hexo n page "pageName" #新建页面

Github Pages设置

创建自己的Github Pages

  1. 注册一个Github账号：Github注册
  2. 在GitHub上新建一个仓库repository，注意仓库名称必须为<你的账户名>.github.io,例如我的是waarm.github.io,这里我的由于已经存在这个名称的仓库，所以无法重名。

  3. 启用自动生成：到我们刚刚新建的仓库页面，点击Settings,找到Launch automatic page generator，点击，继续点击Continue to layout,最后publish page。OK,现在打开浏览器，输入你新建的仓库地址，例如我的是waarm.github.io，就可以看到刚才自动生成的页面了。

SSH Key配置

生成SSH Key

  • 首先我们需要检查你电脑上现有的ssh key：

如果提示：No such file or directory 说明你是第一次使用git。

  • 生成新的SSH Key

  • 然后系统会要你输入密码：

这个密码是你之后将项目部署到到远端(如Github)时需要用到的密码，如果为空的话提交项目时则不用输入，这个设置是防止别人往你的项目里提交内容。(注意：输入密码的时候是没有 *字样的，直接输入就行。)

  • 最后看到这样的界面，表示成功设置ssh key了。

添加SSH Key到Github

在本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置。

  • 资源管理器中找到C:\用户\Admin\.ssh\id_rsa.pub文件,此文件里面内容为刚才生成的密钥。如果看不到这个文件，你需要设置显示隐藏文件。复制文件里面的内容。
  • 登录Github，点击头像下拉框里的Settings,点击左侧的SSH and GPG keys,New SSH key,粘贴刚才复制的内容即可。
